# Google AI Studio: The Scientist (Semantic Validation Protocol)

## 1. Actor Identity and Function
- **Role:** The Scientist / AI Research.
- **Function:** Test prompt routes and validate abstract LangGraph logic before implementing them in the Python source code.
- **Constraint:** This environment is strictly for semantic validation. The LLM is the brain, but it is disconnected from the muscle (Tools). No production code is executed here.

## 2. Tech Lead Actions (Human-in-the-Loop)
Before updating the LangGraph router (`src/core/graph.py`) or Agent prompts, the Tech Lead MUST:
1. Open the Google AI Studio playground.
2. Feed the system prompt of the target agent (e.g., Aequitas Core Supervisor).
3. Paste the Macro/Micro briefing generated by NotebookLM.
4. Execute the "Internal Prompt" below to validate the routing behavior.

## 3. Internal Prompt (Copy and paste into AI Studio)
```markdown
Act as the Graph Router (Aequitas Core Supervisor).
Given the following macro briefing:
[INSERT NOTEBOOKLM BRIEFING HERE]

Define which Sub-Agent (Fisher, Graham, or Marks) should be activated next.
Do not write Python code. Only validate the semantic routing and explain your reasoning based on our Risk Confinement and Zero Numerical Hallucination dogmas.
```
