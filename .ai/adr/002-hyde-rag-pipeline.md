# ADR 002: Hypothetical Document Embeddings (HyDE) RAG Pipeline for Macro Agent

## 1. Context

The Macro Agent must synthesize macroeconomic analysis grounded in official monetary policy
documents: COPOM meeting minutes (Banco Central do Brasil) and FED minutes (US Federal Reserve).
These documents are dense, institutionally-worded, and typically many pages long.

A standard RAG approach — where the user query is directly used as the k-NN search vector —
suffers from *vocabulary mismatch*: a short query such as `"PETR4 macroeconomic analysis"` has
low cosine similarity with paragraphs from formal COPOM resolutions, which use precise monetary
policy language (`"O Comitê decidiu elevar a taxa Selic em 50 pontos-base..."`).

This vocabulary gap degrades retrieval precision and, consequently, the quality of the
synthesized `MacroAnalysis`.

## 2. Decision

We implement **Hypothetical Document Embeddings (HyDE)** as the retrieval strategy for the
Macro Agent, aligned with the MarketSenseAI architecture for institutional document retrieval.

The pipeline has three stages:

1. **Stage 1 — HyDE Generation:** The LLM (`gemini-2.5-flash`, `temperature=0.0`) is prompted
   to generate a dense, hypothetical COPOM/FED announcement text anchored to the target ticker.
   This synthetic document acts as the semantic query vector, eliminating the vocabulary
   mismatch problem. Output is plain text (no `with_structured_output`) to maximize semantic
   density.

2. **Stage 2 — k-NN Retrieval:** The hypothetical document is passed to the injected
   `VectorStorePort.search_macro_context(hyde_text, as_of_date, top_k=5)`. The vector store
   returns the top-5 most semantically similar real documents from the indexed corpus that are
   valid for the active point-in-time boundary.

3. **Stage 3 — Grounded Synthesis:** The LLM synthesizes a validated `MacroAnalysis` using
   `with_structured_output(MacroAnalysis)`, grounded in the retrieved chunks. All `source_urls`
   are populated deterministically from retrieval metadata — never hallucinated by the LLM.

The `VectorStorePort` interface (defined in `src/core/interfaces/vector_store.py`) provides the
DIP boundary. `NullVectorStore` guarantees Controlled Degradation in offline/local environments
by returning an empty list, triggering the LLM to fall back to internal knowledge with all
`Optional[float]` fields set to `None`.

The retrieval contract is now explicitly time-aware: qualitative retrieval must receive
`as_of_date` from `AgentState` so that contextual evidence respects the same anti-look-ahead
boundary enforced across quantitative tooling and backtesting.

## 3. Consequences

**Positive:**
- Retrieval precision is substantially improved for dense institutional corpora.
- Source traceability is enforced at the architecture level: `source_urls` are injected
  from retrieval metadata, never generated by the LLM (Zero Hallucination dogma preserved).
- HyDE is language-agnostic; prompting in pt-BR aligns with the indexed corpus language.

**Negative:**
- +1 LLM call per Macro Agent execution (Stage 1), increasing API costs and latency.
- Free-Tier throttle (`time.sleep(15)` before Stage 1, `time.sleep(10)` before Stage 3) adds
  25 seconds of wall-clock latency per run — acceptable for offline batch analysis, not for
  real-time applications.
- The HyDE prompt is tightly coupled to the corpus language (pt-BR). If the index is
  migrated to an English corpus, the Stage 1 prompt must be updated accordingly.
